# WORKFLOW — работа над Zubrilka на двух ПК

Проект синхронизируется **только через GitHub** (`origin` → https://github.com/amelnikov777/Zubrilka, ветка `main`).
Оба компьютера обмениваются кодом через него — напрямую они не связаны.

```
  Рабочий ПК (VS Code)                        Домашний ПК (Visual Studio)
   разработка + тесты                          сборка APK + тест на Android
          │                                              │
          │ git push  ──────►  GitHub  ◄──────  git pull │
          └──────────────────────────────────────────────┘
```

**Золотое правило:** начинаешь с `git pull`, заканчиваешь `git commit` + `git push`.

---

## 🏢 На работе (VS Code) — разработка

```bash
git pull                       # 1. забрать свежий код ПЕРЕД работой
# ... пишешь и правишь код, тестируешь логику ...
git add -A                     # 2. в конце — сохранить
git commit -m "что сделал"
git push                       # 3. отправить на GitHub
```

В VS Code то же самое доступно через панель **Source Control** (Pull / Commit / Push).

---

## 🏠 Дома (Visual Studio) — сборка и тест на устройстве

```bash
git pull                       # 1. забрать код с работы
```

2. Собрать / запустить на телефоне:
   - **Visual Studio:** выбрать вверху подключённое Android-устройство → нажать **▶ (F5)**.
     Студия сама соберёт и зальёт приложение.
   - **Или из терминала:** `dotnet build -c Release`
3. Протестировать на реальном Android-устройстве (USB + включённая «Отладка по USB»).
4. Если правил код дома — тоже `git add -A && git commit -m "фикс" && git push`.

## ⚙️ Требования окружения (уже настроены на домашнем ПК)

Чтобы `dotnet restore` / `dotnet build` прошли, нужны:

- **.NET 10 SDK** (проект таргетит `net10.0-android`; на .NET 8/9 будет ошибка `NETSDK1139`).
- **MAUI workload:** `dotnet workload install maui-android`.
- **Android SDK Platform 36** (иначе `XA5207: Could not find android.jar for API level 36`).
  Установка требует прав администратора:
  ```
  dotnet build -t:InstallAndroidDependencies -f net10.0-android -p:AcceptAndroidSDKLicenses=true
  ```

> Предупреждения `XA4301` про дубль `libe_sqlite3.so` при сборке — безобидны, это не ошибка.

---

## ⚠️ Чтобы не сломать синхронизацию

1. **Всегда `push` на работе и `pull` дома** — самая частая ошибка: уехал, а код не запушен.
2. **Не редактируй один файл на обеих машинах без синхронизации** — иначе конфликты слияния.
3. **`bin/` и `obj/` не коммитятся** — пересобираются на каждой машине сами (должны быть в `.gitignore`).
